## Scope

A package's 30-day install count tells you it's `ca-certificates`. Velocity tells you what's actually getting adopted right now. v0.4.0 introduces:

- **Velocity scoring (A, free)** — derive an "above/below annual monthly average" index from the three rolling windows formulae.brew.sh already publishes. Sort by velocity by default; the heat (🔥) and cool (❄️) badges replace the "every row is ca-certificates" leaderboard.
- **`install-on-request` switch** — pull the secondary endpoint in parallel with the primary `install` endpoint and surface the user-initiated install count. Strips dependency-pull noise from the signal.
- **Opt-in history endpoint (B)** — `brew-browser.zerologic.com/trending-history/*`. Distinct trust boundary (our infra vs. Homebrew first-party), so gated by a new `enhanced_trending_enabled` toggle in Settings → Network. Paranoid mode hard-locks it off.
- **Seed-from-rolling-windows trick** — the day the collector turns on, it derives three historical buckets per package from current 30d/90d/365d subtractions. Charts have data on day 0; granularity grows from daily snapshots thereafter.

This task record covers the backend pieces; frontend, collector, docs, and Caddy come in their own commits / task records.

## Decisions

- **D1 — Endpoint domain:** `brew-browser.zerologic.com/trending-history/*` (subpath on existing vhost), not a new `trending-history.zerologic.com`. Reuses the Caddy + cert already serving the updater manifest. Same DNS-level trust boundary; only feature-level differentiation.
- **D2 — GitHub mirror of nightly JSON:** deferred to v0.5+. The auditability is nice-to-have, not blocking.
- **D3 — Sort default:** velocity desc, with an inline mini-sparkline per row when enhanced trending is on (mirrors star-history.com list view). Index blob carries the compact sparkline so the trending list renders from one fetch.
- **D4 — Sparkline empty state when toggle is off:** passive — only present in Settings → Network. No placeholder, no banner. Detail panel simply omits the sparkline area when the toggle is off.
- **D5 — Velocity computation:** server-side in `trending::client::fetch` → joined in `commands::trending::trending_fetch` from the cached 30d/90d/365d. Frontend never needs to know the formula.
