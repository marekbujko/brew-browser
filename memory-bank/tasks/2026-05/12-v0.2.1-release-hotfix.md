## What landed

### Primary fix: macOS Keychain prompt on every launch
- The v0.2.0 fix from `e04dbff` added an eager `void github.loadStatus()` to `+layout.svelte`'s `onMount`. That call probes Keychain via `keyring::get`, which prompts the user when the binary signature doesn't match an existing item's ACL.
- Fresh v0.2.0 installs (new signature) saw `"brew-browser wants to use your confidential information stored in dev.openbrew.browser"` prompt on every launch — even users who'd never touched a GitHub feature. Worse, it trained users to dismiss the prompt without context.
- **Fix:** removed the eager call. Made `requireGithubSignIn(actionLabel)` async; it now lazy-probes the Keychain on the first GitHub-action click (Star / Watch / File-issue). Settings → GitHub still hydrates status on panel mount.
- **Net effect:** macOS prompt only fires when user is actively trying to use the token — contextual moment it was designed for.
