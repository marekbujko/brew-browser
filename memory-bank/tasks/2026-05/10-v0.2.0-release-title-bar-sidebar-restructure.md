## Scope

Structural UI overhaul triggered by user direction across multiple interactive iterations: native macOS title bar with traffic-light alignment, collapsible sidebar with persistent type-ahead search, info popovers replacing AI badges and "Wrong?" links, intercept-on-action GitHub sign-in (no static "Sign in" prompts), Dashboard moved into the nav as a normal item, native-square app icon (Tahoe-clean).

## What landed

### Native macOS title bar
- New `<header class="titlebar">` in `+page.svelte` spans the window above the main split — the Mac unified-toolbar pattern (Mail / Notes / Messages)
- **36 px tall**, same `--color-surface-raised` chrome as sidebar — forms one continuous L-shaped frame
- `tauri.conf.json` gains `trafficLightPosition: { x: 14, y: 20 }` so macOS-rendered traffic lights vertically center on the same axis as the toggle and the title
- **Page title** (`ui.pageTitle`, derived from `ui.section` via a `SECTION_TITLES` lookup) lives in the title bar with absolute positioning + CSS variable left-offsets so it sits just past the sidebar divider
- Each pane's `<h1>` removed from `panel-head`; where the head had no remaining content (Dashboard, Discover) the entire `<header>` dropped. Library / Trending / Snapshots / Services / Activity kept their heads as thin secondary toolbars

### Title-bar right cluster (new `TitlebarControls.svelte`)
- Theme dropdown (single icon → 3-item popover Light/Dark/System) + Settings gear + pink-filled Donate heart
- Pill-grouped with hair-line dividers
- Background uses `--color-surface` (panel-body gray) — softer than dark sunken bg
- Right edge aligns with main panel's right padding (`var(--space-4)`)
- Cluster nudged 1px below center to optically match the macOS traffic lights

### Collapsible sidebar + type-ahead search
- New `ui.sidebarCollapsed` state + `toggleSidebarCollapsed()` + `loadSidebarCollapsedFromStorage()` — persists to `brew-browser:sidebar-collapsed` localStorage
- 200px → 56px transition. Nav items go icon-only with 14px badge overlay
- Sidebar toggle position adapts via CSS variables: `left: 168px` (inside sidebar's right edge) when expanded, `left: 84px` (next to traffic lights) when collapsed
- **Dashboard becomes the first nav item** (`LayoutDashboard` icon, `⌘0`) — menu bar already identifies the app; brand text redundant
- New persistent **type-ahead search input** at sidebar top (no separator below). Debounced 300ms → `brew_search`. Top 7 results dropdown with `[name] [kind pill] [installed badge]`. ArrowUp/Down + Enter + Esc keyboard support. "See all results in Discover →" link. Hidden in collapsed mode.
- Theme + Settings + Donate **removed** from sidebar footer (moved to title-bar cluster)

### Info popovers (new `InfoButton.svelte`) replace AI labels + "Wrong?" links
- Hover-activated (120ms open / 180ms close delays); focus-supported for keyboard a11y; click toggles for touch
- `position: fixed` with viewport-aware top/left — escapes ancestor `overflow: hidden` (was getting clipped by detail panel's `.body`)
- Auto-closes on scroll / Esc / outside-click
- No `title` attribute on trigger (popover IS the tooltip; native title bubble would collide); `aria-label` carries accessible name; `aria-expanded` + `aria-haspopup="dialog"` signal state
- **All "Wrong?" links gone** (Categories, Tags, Summary, Why install this?, Similar packages)
- **All "AI-enriched" sparkle badges gone** — provenance now one tap away via (i) on each enriched field
- Body text: *"Generated offline at build time by Claude Haiku 4.5 — no network or LLM calls happen while you use brew-browser. Open an issue if X looks off and we'll fix it in the next release."*
- Summary's (i) sits **inline at the end of the summary text** (with `&nbsp;` so it doesn't wrap)

### Intercept-on-action GitHub flow
- Removed the static "Sign in via Settings → GitHub to star, watch, or file issues." paragraph
- Star / Watch / File issue buttons always render when GitHub stats card is visible, regardless of sign-in
- New `requireGithubSignIn(actionLabel)` helper: signed-in → proceeds; signed-out → `ui.openSettings("github")` + toast hint
- **Intent-based discovery** — sign-in prompt only appears when user actually wants to act
- Settings deep-link plumbing: `SettingsSection` type promoted to `types.ts`; `ui.openSettings(section?)` accepts optional target; `ui.settingsInitialSection` cleared on close
- **GitHub OAuth `client_id` set live** to `Ov23liJZKbvrSBuiOPkT` — Device Flow client_ids are public per RFC 8628 §3.1, safe to commit
- Sign-in friendlier error using `brewErrorMessage(e)` (was `e.code`); DeviceFlowModal drops redundant `toast.error` (modal renders message inline)

### Other UI fixes
- **License-mismatch row** wraps as prose (whole sentence in one `<span>`, AlertCircle as the only non-shrinking sibling) instead of fragmenting into a 2-column grid
- **EmptyState vertically centered** — added `min-height: 100%` to `.empty`; every empty state across the app (Library, Discover, Snapshots, Trending, etc.) now sits in pane center
- **Snapshots inline CTAs removed** — `New Snapshot` / `Import Brewfile…` in empty state were duplicates of panel-head actions
- **Selected-row state persists across panes** — Library, Trending, Discover (both variants), Services, Dashboard outdated bind `selected={…}` to `ui.selectedPackage`
- **Chip-filter clears on cross-pane navigation** — `ui.setSection()` calls `discover.clear()` when section actually changes; deeplink callers reordered to call setSection first then selectOnly

### Native-square app icon (Tahoe-clean)
- `docs/icon/brew-browser.svg` re-authored as full-bleed 181×181 square (no pre-rounded outer squircle)
- All icon sizes regenerated via `npm run tauri icon`
- Fixes macOS Tahoe double-squircle artifact (the OS auto-mask was exposing transparent corners in the previous pre-rounded source as "white corners")

## Notes / decisions

- **Largest UX overhaul in the project** — happened in a single ~2.5h interactive session with the user, ~10 round-trips of "let me see it" → tweak → "tighter please" → tweak.
- **`!important` on `.panel-head` baseline** + CSS variables driving toggle/title positions = clean transitions without JS.
- **Intercept-on-action discovery** is a posture decision worth calling out: the app no longer evangelizes its GitHub feature. Users discover it only when they try to use it. Less clutter, less interruption.
- **Native-square icon** fixed a Tahoe-specific regression the user noticed — the OS auto-masks every app icon now, and pre-rounded source PNGs produce double-corner artifacts. Worth knowing for any other Mac app launch in 2026.
- v0.2.0 .dmg shipped at 18:06 UTC. Within ~30 min, three GitHub-auth bugs surfaced that required a hotfix (`e04dbff`) and v0.2.1 release the same day. See task 11.
