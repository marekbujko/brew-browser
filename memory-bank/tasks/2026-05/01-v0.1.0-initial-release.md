## Scope

Birth of brew-browser — a native macOS GUI on top of Homebrew, built phase-by-phase before the first git commit. Eight sequential feature phases, then a security audit pass, all bundled into the v0.1.0 release artifact.

## What landed

**Phase 0 — Scaffold.** Tauri 2 + SvelteKit + Svelte 5 + Rust shell. Project structure, Cargo.toml + tauri.conf.json baseline, CSP defaults, capability allowlist (minimal: core:default, opener:default, dialog:allow-open, dialog:allow-save).

**Phase 1 — Read-only Homebrew browser.** `brew_list_installed`, `brew_info` IPC commands. PackageDetail panel. Library view rendering installed formulae + casks with kind pills.

**Phase 2 — Search Homebrew index.** `brew_search` IPC against `brew search`. Discover view with debounced search input (300ms) → results list. Categories UI deferred to Phase 9a.

**Phase 3 — Install / uninstall / upgrade w/ streaming.** `tokio::process::Command` invocations with `tokio_stream` stdout/stderr channels. Live event streaming via typed Tauri channels (`JobEvent::Started | Output | Finished`). ActivityDrawer mounted at bottom of window. No `tauri-plugin-shell` — every command built in Rust from typed enums.

**Phase 4 — Brewfile snapshot / restore.** `brewfile_export`, `brewfile_import`, `brewfile_restore` commands shelling to `brew bundle dump/install/check`. Path sandboxing on import/export (forbidden-prefix list + 1 MiB size cap). Friendly error mapping for the known-broken `brew bundle check` upstream bug.

**Phase 5 — Polish + build artifact.** First unsigned `.dmg`. Window vibrancy via `window-vibrancy`. Native macOS title bar (`titleBarStyle: "Overlay"`, `hiddenTitle: true`, `transparent: true`). Default 1100×720 window, minimums 800×500. Signing pending Developer ID cert install (deferred to commit `cb60e4a`).

**Phase 6 — Trending tab.** `trending_fetch` against `formulae.brew.sh/api/analytics`. 60-minute in-process cache (TTL configurable later in Phase 12b). 30/90/365-day windows. Sortable columns added in Phase 9b.

**Phase 7 — Cask icons (installed).** Backend extracts icons from installed `.app` bundles. 24-hour persistent disk cache (`icon-cache/`). PackageRow renders the 24×24 icon next to the cask token.

**Phase 8 — Cask icons (homepage cascade).** For uninstalled casks with a `homepage`, the backend probes (in order): `/apple-touch-icon.png`, `<meta og:image>` parsed from homepage HTML, `/favicon.ico`. Per-cask weekly probe cap. Result cached 7 days including misses. **SSRF defense:** link-local, loopback, RFC1918, cloud-metadata IPs rejected pre-flight and on every redirect hop.

**Initial security audit (pre-commit).** Tool battery: cargo audit (0 vulns), cargo deny (advisories/bans/licenses/sources ok), npm audit --omit=dev (0), semgrep (security-audit + OWASP-10 + Rust + TS, 113 rules, 0 findings), gitleaks (0). 16 findings from manual review all verified-fixed with passing tests. Verdict: **READY-FOR-SCRUTINY**.

## Notes / decisions

- **No `tauri-plugin-shell`** — every brew invocation built in Rust from typed enums. Frontend cannot construct arbitrary shell commands.
- **No telemetry, no accounts, no analytics, no third-party pixels** — posture established here, defended in every subsequent phase.
- **MIT licensed.** No CLA, no EULA.
- **Pre-init phase work** was done in a single multi-hour session before `git init` was run. The first commit captures all 8 phases at once.
- Catalog data ships as a placeholder; real catalog bundling is Phase 12a (~24 hours later).
- Apple Silicon only at v0.1.0. Intel + Universal binaries deferred.
- `brew tap` for one-line install on the roadmap (not in v0.1.0).
